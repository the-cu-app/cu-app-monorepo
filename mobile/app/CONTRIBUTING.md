# Contributing to CU.APP

Thank you for your interest in contributing to CU.APP! We welcome contributions from the community.

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/yourusername/cu-app.git`
3. Install dependencies: `flutter pub get`
4. Set up your Supabase configuration (see README.md)

## Development Guidelines

### Code Style
- Follow Dart/Flutter conventions
- Use meaningful variable and function names
- Add comments for complex logic
- Format code with `dart format`

### Commit Messages
- Use clear, descriptive commit messages
- Start with a verb (Add, Fix, Update, etc.)
- Reference issues where applicable

### Security
- Never commit API keys or secrets
- All sensitive data should be in environment variables
- Follow security best practices

## Pull Request Process

1. Create a feature branch from main
2. Make your changes
3. Test thoroughly
4. Update documentation if needed
5. Submit a pull request

## Questions?

Feel free to open an issue for discussion or reach out to the maintainers.
